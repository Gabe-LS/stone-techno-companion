=== CLAIM B (HIGH): webpush() called with no timeout → thread-pool exhaustion ===
In server/chat_ws.py ~465-479 and server/api.py ~361-374, webpush(...) is invoked via asyncio.to_thread with no timeout= argument; pywebpush's underlying requests.post defaults to no timeout (blocks indefinitely). Combined with the endpoint SSRF, an attacker registering a subscription pointing at a server that accepts but never replies causes that thread-pool worker to hang forever. The default asyncio executor thread pool is shared app-wide, so several such subscriptions receiving pushes can exhaust the pool and stall unrelated request handling. Verify no timeout is passed and the executor is shared.
